# Контекст проекта: LocalScript (MTS True Tech Hack 2026)

Ты — мой экспертный соавтор и технический консультант. Мы участвуем в хакатоне, задача которого — создать **LocalScript**: локальную агентскую систему для генерации и отладки Lua-кода.

### Суть проекта
Система должна работать **полностью автономно (offline)** на обычном ноутбуке. Мы используем локальные LLM (8B параметров, например, DeepSeek-Coder или Llama 3) через Ollama. Главная фишка — **Agentic Loop**: система не просто пишет код, а запускает его в изолированной песочнице, ловит ошибки и сама их исправляет в цикле, пока код не заработает.

### Технический стек
* **Языки:** Python (основной), Lua (целевой).
* **Инструменты:** Ollama (инференс LLM), LangGraph/LangChain (логика агента), Docker (песочница), FastAPI (бэкенд), ChromaDB/FAISS (RAG).
* **Протокол обмена:** Строгий JSON-контракт между компонентами (вход: `{"code": "...", "timeout": 5}`, выход: `{"status": "...", "output": "...", "error_msg": "...", "exit_code": int}`).

### Структура команды и задачи

#### 1. Бэкенд-разработчик (Инфраструктура)
**Рабочие директории:** `sandbox/` и `src/api/`
* Создание Docker-песочницы для безопасного исполнения Lua.
* Написание Python-валидатора (захват stdout/stderr и их упаковка в стандартный JSON-ответ).
* Разработка API на FastAPI и общая оркестрация сервисов.

#### 2. AI-инженер [Strong] (Логика и Граф)
**Рабочая директория:** `src/agents/`
* Построение архитектуры агента в LangGraph (циклы самокоррекции).
* Prompt Engineering: создание системных промптов для стабильной работы 8B моделей.
* Логика обработки ошибок (парсинг JSON-ответа от песочницы и превращение `error_msg` в понятную инструкцию для LLM).

#### 3. AI-инженер [Standard] (RAG и Качество)
**Рабочие директории:** `src/rag/` и `tests/`
* Разработка RAG-системы: сбор и индексация документации Lua.
* Настройка Retrieval-логики (поиск релевантных кусков кода под запрос).
* Подготовка Eval-набора (тестовые задачи для проверки качества работы агента).
