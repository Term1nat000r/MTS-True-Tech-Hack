# Проект: LocalScript (Локальная агентская система для Lua)

## 🏗 Тематические блоки работы

### 1. Agentic Logic (Ядро и мозг системы)
* **Prompt Engineering `[Изолированно]`:** Написание и отладка системных промптов в интерфейсе Ollama.
* **Цикл самокоррекции `[Изолированно]`:** Проектирование логики графа (через LangGraph). Можно тестировать на ручном вводе ошибок.
* **Парсинг ответов `[Изолированно]`:** Написание регулярных выражений для очистки Lua-кода от Markdown-разметки.
* **Управление контекстом `[Зависимость]`:** Требует понимания, какой объем логов будет приходить из песочницы.

### 2. Sandbox & Execution (Безопасная среда)
* **Интеграция с Lua `[Изолированно]`:** Установка и настройка интерпретатора, проверка базовых команд запуска скриптов через Python (`subprocess`).
* **Изоляция (Sandboxing) `[Изолированно]`:** Написание Dockerfile и скриптов запуска контейнера.
* **Runtime-валидатор `[Зависимость]`:** Нужно согласовать с AI-инженером формат вывода ошибок (JSON или сырой текст), чтобы агент их понимал.

### 3. Knowledge Base & RAG (Память и контекст)
* **Dataset & Parsing `[Изолированно]`:** Сбор документации Lua и преобразование её в Markdown/текст для базы.
* **Векторная база данных `[Изолированно]`:** Настройка ChromaDB/FAISS и генерация эмбеддингов для собранных данных.
* **Retrieval-логика `[Зависимость]`:** Нужно интегрировать поиск в промпт AI-инженера.

### 4. Backend & Orchestration (Инфраструктура)
* **Инференс-прослойка `[Изолированно]`:** Настройка API-клиента для связи с локальной Ollama.
* **API Layer `[Изолированно]`:** Создание скелета FastAPI (эндпоинты `/generate`, `/status`).
* **Сборка системы (Оркестрация) `[КРИТИЧЕСКАЯ ЗАВИСИМОСТЬ]`:** Финальное соединение AI-агента, RAG и Docker-песочницы.

## 👥 Распределение ролей

### 👤 Участник 1: Бэкенд-разработчик (Инфраструктура)
1.  **`[Изолированно]`** Создание Docker-песочницы.
2.  **`[Изолированно]`** Настройка сервера FastAPI.
3.  **`[Зависимость]`** Связка песочницы с основным бэкендом (после готовности Docker-скриптов).

### 👤 Участник 2: AI-инженер Strong (Логика и Граф)
1.  **`[Изолированно]`** Написание системных промптов.
2.  **`[Изолированно]`** Построение архитектуры агента в LangGraph (на "заглушках").
3.  **`[Зависимость]`** Финальная прошивка логики под реальные ошибки из песочницы.

### 👤 Участник 3: AI-инженер Standard (RAG и Качество)
1.  **`[Изолированно]`** Сбор и индексация данных (RAG).
2.  **`[Изолированно]`** Подготовка тестов (Eval) — набор задач на Lua, на которых будете проверять агента.
3.  **`[Зависимость]`** Тюнинг поиска под конкретные запросы, которые генерирует "Strong" инженер.
